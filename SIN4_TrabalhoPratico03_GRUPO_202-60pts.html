1Descrição

O presente trabalho será em grupo (uma mesma pessoa não pode estar em dois ou mais grupos). Cabe aos alunos se organizarem a respeito dos grupos.

2O trabalho

Nessa seção será apresentado o que se espera que cada grupo execute neste trabalho sendo o processo descrito a seguir.

Cada grupo deverá implementar uma API usando persistência Spring JPA (usando H2) conforme o exemplo apresentado no projeto restappjpa.

A API deve ter ao menos UMA entidade a ser persistida.

A APP de consumo deve possuir interface gráfica WEB (conforme exemplo do projeto AppCarrosHTML) ou compatível com com o SO Android (conforme exemplo do projeto) e que efetue as atividades de CRUD (INSERIR, ATUALIZAR, EXCLUIR e LISTAR) sobre uma ou mais entidades escolhidas pelo grupo segundo o especificado neste documento.

A entidade deve ser conveniente ao domínio especificado tais como: Cliente, Produto, Carro etc.

A entidade deve ter pelo menos 4 (quatro) atributos (nome, idade, endereço e cpf por exemplo)

Os atributos da entidade devem ser de, no mínimo, 2 (dois) tipos diferentes (String e int por exemplo).

3Entrega do trabalho

Neste capítulo será apresentado o que se espera que cada grupo execute neste trabalho sendo descrito a seguir.

3.1Envio

  • •A API REST deverá ser feita utilizando a IDE Netbeans / MAVEN 

    • ◦Exclua a pasta TARGET antes de compactar o projeto para envio 

  • •A APP de consumo deverá ser feita usando o Netbeans ou Android Studio (dependendo do tipo de projeto escolhido) 

    • ◦No caso de projeto web compacte a pasta completa do projeto 

    • ◦No caso de projeto Android, compacte apenas a pasta SRC 

Ambos os artefatos (API REST e APP de consumo) devem ser enviados pelo Moodle em um ÚNICO ARQUIVO ZIP.

3.2Entrega

As pastas (da API REST e da APP de consumo) deverão ser compactadas em um ÚNICO arquivo ZIP.

Esse arquivo deve ser nomeado seguindo o seguinte critério:

  • SIN4BD_202_TR03_nome01.zip 

    • oo(para grupos de 1 aluno) 

  • SIN4BD_202_TR03_nome01_nome02.zip  

    • oo(para grupos de 2 alunos) 

Onde nome01 (nome02) são os nomes dos integrantes do grupo. Por exemplo, para os alunos Paulo Roberto e Marina Ferreira o nome do arquivo será:

SIN4BD_202_TR03_PauloRoberto_JoseAlcantara_MarinaFerreira.zip

O arquivo deverá ser postado no moodle (http://moodle.ucv.edu.br/moodle/login/index.php) da disciplina por um integrante do grupo (APENAS). Grupos com mais de um envio não serão avaliados!

A data limite de envio é 17/12/2020 até as 23:55.
